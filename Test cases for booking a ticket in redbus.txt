Topic: Manual Testing
Write few test cases for booking a ticket in redbus

1. Verify that the user is able to search for available buses by providing the source and destination locations, and the date of travel.

2. Verify that the user is able to select the desired bus from the available options and proceed to the seat selection page.

3. Verify that the user is able to select the desired seat(s) and proceed to the passenger details page.

4. Check if that the user is able to enter the passenger details (name, age, gender, etc.) and proceed to the payment page.

5. Verify that the user is able to select the desired payment method and make the payment successfully.

6. Now verify that the user is redirected to a confirmation page after the payment is successful and the ticket details are displayed correctly.

7. Verify that the user is able to view and download the ticket in PDF format from the confirmation page.

8. Verify that the user is able to cancel the ticket before the journey date and receive a refund if applicable.

9. Check if that the user is able to view the booking history and track the status of the booked tickets.

10. Verify that the user is able to contact customer support in case of any issues related to the booking or travel.
